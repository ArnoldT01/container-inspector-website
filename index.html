Features

Everything in one window

Stop switching between Docker CLI, container logs, and GitLab's UI. Container Inspector brings it all together.

⬡

Deep container inspection

Overview, CI/CD variables, environment, mounts, and networking — all organised into logical tabs for every running container.

◈

Live GitLab CI monitoring

See running jobs grouped by triggerer with real-time status. Select any job for a full breakdown of pipeline, MR, and runner details.

🔗

Job ↔ container correlation

Automatically detects when a local GitLab runner container is executing a CI job and surfaces a ● local badge — no guesswork.

🔔

Smart notifications

Set your name and a runner to watch. Get notified the moment your job starts on that machine, even if the app is in the background.

🔒

Secure token storage

Your GitLab personal access token is encrypted with the OS keychain (or AES-256 fallback). Never stored as plaintext on disk.

🕵️

Environment variable inspector

Browse all env vars with sensitive keys auto-masked. One-click reveal and copy-to-clipboard for any value.

Views

Five views, zero clutter

Navigate between Docker containers, GitLab jobs, request logs, support, and about in a clean collapsible sidebar.

⬡

Local — Docker containers

Real-time container list with auto-refresh every 5 seconds. Filter to GitLab runners only. Inspect any container across four detail tabs.

Overview CI/CD env vars All env vars Mounts & Network
◈

GitLab — CI/CD jobs

Live feed of running jobs, grouped by user, refreshed every 15 seconds. Expand any job for full pipeline, MR, and runner detail.

Job overview Pipeline info MR metadata Runner detail
☰

Logs — API request inspector

Every IPC call and API request the app makes is logged with status, duration, and full response. Invaluable for debugging connectivity issues.

Request list Duration Response body

Settings

Configure your GitLab token (auto-fills your name from the API), project ID, the name to watch for, and your target runner — all persisted securely.

Get started

Up and running in minutes

No config files, no CLI setup. Just download, open, and connect.

1

Download and open

Grab the latest release from GitHub. Runs on Windows, macOS, and Linux via Electron.

2

Add your GitLab token

Open Settings, paste your personal access token, and enter your project ID. Your token is encrypted immediately on save.

3

Watch your runner

Type your name and pick a runner from the dropdown. You'll get a notification the moment one of your jobs starts on that machine.

4

Inspect away

Switch between Local and GitLab views. Click any container or job to drill into its full CI/CD context, environment, and runner detail.

Requirements

What you need

Container Inspector connects to two things — your local Docker daemon and the GitLab API.

🐳

Docker Desktop

Docker must be running locally so the app can connect via the Docker socket and list your containers.

◈

GitLab access token

A personal access token with read_api scope on gitlab.com or your self-hosted instance.

📋

GitLab project ID

Found on your project's main page in GitLab. Needed to fetch jobs and pipeline data for your project.
